Git

運作流程

開啟資料夾-建立檔案-建立數據庫-(init)-工作目錄-(add)-索引-(commit)-本地數據庫-(push)-遠端數據庫-(clone)-回傳至本地數據端
須配合Gitub使用

環境設定(只需設定一次)

git config --global user.email"使用者信箱帳號"(使用者信箱)

git config --global user.name"使用者名稱"(使用者名稱)

git config --global user.name"使用者名稱"(使用者名稱)

基礎操作

cd 路徑，範例 : cd/test/css(移動至css資料夾內，可直接拖曳資料夾進git視窗介面中)

cd ..(回上一層)

ls(展開當前所在位置的所有資料夾）

touch 檔案名稱，範例touch index.html(建立新資料夾)

mkdir 資料夾名稱，範例 :mkdir css(開啟css的資料夾)

數據庫建立

git init(確認當前位置正確後輸入，出現master字樣完成)

加入索引

git add .(新增所有檔案至索引)

git add 檔名，範例 : git add test.txt(加入某個檔案至索引，需加入檔名與副檔名)

git status(檢視狀態)

狀態註解 : modified(經過編輯)

忽略檔案

gitignore "忽略此檔案內所記錄的檔案"(將含副檔名的檔案輸入，並儲存)

在此檔案內輸入:
檔名.副檔名(忽略此檔案)
*.副檔名(忽略此副檔名的全部檔案)
資料夾名稱/(忽略此資料夾內的所有檔案)
此檔案也需要加入索引、以及提交更新

取消索引

git reset HEAD(全部檔案取消索引)

git reset HEAD 檔名，範例 : git reset HEAD text.txt(全部檔案取消索引)

還原檔案

git checkout(單一檔案到最新commit狀態)

git reset --hard(還原工作目錄與索引至最後一次commit狀態)

將遠端回傳至本地數據

git clone github上的網址(下載檔案)

將本地數據上傳至遠端數據

git push(需先經過git clone流程)

git push -set-upstream origin 分支名稱(上傳分支)

git push origin 分支名稱，範例 : git push origin master(origin為預設遠端主機名稱)

分支

HEAD : 目前所在的位置指標

git branch(瀏覽目前分支)

git branch 名稱(創立一個新分支)

git branch -d 名稱(移除某個分支)

git checkout前四碼(回到某個commit狀態)

git checkout 名稱(將HEAD移動到此分支)

git checkout master(回到預設的分支)

合併

git merge 分支名稱(與master進行合併)

當分支與master記錄點不同時，可合併為同一紀錄點
衝突解決

衝突顯示

<<<<<<< HEAD
      <h1>......</h1>
            =======(等號上方是原本branch的程式碼，等號下方是要合併分支的程式碼)
      <h1>......</h1>
>>>>>>> 分支名稱 完成後需要手動add與commit
標籤

git tag(查詢標籤)

git tag -n(查詢詳細標籤)

git tag -d 標籤名稱(刪除標籤)

git tag -am "備註內容" 標籤名稱(新增標籤備註)

git checkout 名稱(將HEAD移動到此分支)

git checkout 標籤名稱(將HEAD切換到該標籤)

暫存

git stash(暫時儲存當前目錄)

git stash list(瀏覽git stash列表)

git stash pop(還原暫存)

git stash drop(清除最新暫存)

git stash clear(清除全部暫存)

git status無法查詢此類型狀態
臨時處理其他專案時，檔案尚未完成，可使用此功能
推送分支至遠端

git remote(查詢遠端數據庫)

git remote rename origin 修改的名稱(重新命名遠端數據庫)

origin為預設遠端主機名稱
下載遠端數據庫

git pull(拉資料至工作目錄)

當github更新紀錄與本地端不符時
需先git pull下載遠端資料
才能進行git push
Github 線上平台

New repository(右上角 + 號的下拉選單，創立遠端資料庫)
